Addax es una librería minimalista de javascript. Si sabes usar jQuery, sabes usar Addax

Mientras jQuery no cubre los objetivos de este diseño, la API esta a la altura de su homólogo. El objetivo es tener una librería modular de 4-8k - que descargue y ejecute rápido, con una API familiar y versátil.

Addax es software libre y se distribuye bajo la promoción de negocios.

Download Addax

El código por defecto incluyen los siguiente módulos:
Core, Ajax, Events, Detect, Dom, Templating.

  • addax.js v1.0 (desarrollo) – 12.3k sin comprimir
  • addax.min.js v1.0 (producción) – 5.0k comprimido

Incluye Addax con un tag script antes de finalizar tu página. 

...
<script src="addax.min.js"></script>
</body>
</html>

Si tu necesitas que tenga soporte con IE, puedes hacer un fall back 

<script>
document.write('<script src=' +
('__proto__' in {} ? 'addax' : 'jquery') +
'.js><\/script>')
</script>

Tenga en cuenta que algunas de las características de Addax se dirigen específicamente a los navegadores móviles, ya que el objetivo original del proyecto era proporcionar específicamente una alternativa más ligera a jQuery para la web móvil.

Addax es una buena opción para el navegador (por Safari y Chrome) y desarrollar puntos de vista basados ​​en HTML dentro de los marcos de aplicaciones nativas, como PhoneGap.

En resumen, se espera que Addax pueda trabajar en todos los navegadores modernos.

Crear plug-ins

Los plug-ins pueden ser creados agregando una propiedad a addax: 

;(function($$){
  $$.extend($$.fn, {
    foo: function(){
      // Esto hace referencia a la actual colección de Addax.
      // Retornamos el objeto.
      return this.html('bar')
    }
  })
})(addax)

Métodos del Core

$()

$.find(selector) ⇒ collection
$(<Addax collection>) ⇒ same collection
$(<DOM nodes>) ⇒ collection
$(htmlString) ⇒ collection
$(htmlString, attributes) ⇒ collection [v1.0]

Crear una colleción de objetos Addax mediante la realización de un selector CSS, envolviendo nodos del DOM o creando elementos desde una cadena en HTML

Una colección es un objecto matriz a la cual se le encadenan métodos para manipular los nodos del DOM que hace referencia. Todos los métodos de la presente documentación son los métodos de recolección, excepto los que actúan directamente sobre el objeto $ (addax), como $.extend

$('div')  //=> todos los elementos div de la página
$('#foo') //=> elementos con ID "foo"

// Crear elementos:
$("<p>Hello</p>") //=> Un nuevo elemento P

$$.extend

$$.extend(target, [source, [source2, ...]]) ⇒ target
$$.extend(true, target, [source, ...]) ⇒ target [v1.0]

Extender un objeto con todas las propiedades del objeto padre

var target = { one: 'patridge' },
    source = { two: 'turtle doves' }

$$.extend(target, source)
//=> { one: 'patridge',
//     two: 'turtle doves' }

addClass

addClass(name) ⇒ self
addClass(function(index, oldClassName){ ... }) ⇒ self

Agrega el nombre de una clase para la colección de los elementos.

append

append(content) ⇒ self

Agregar contenido al DOM dentro de cada elemento individual de la colección. El contenido puede ser una cadena HTML, un nodo DOM o una matriz de nodos.

$('ul').append('<li>new list item</li>')

prepend

prepend(content) ⇒ self

Agregar contenido al DOM antes del contenido de cada elemento individual de la colección. El contenido puede ser una cadena HTML, un nodo DOM o una matriz de nodos.

$('ul').prepend('<li>new list item</li>')

attr

attr(name) ⇒ string
attr(name, value) ⇒ self
attr(name, function(index, oldValue){ ... }) ⇒ self
attr({ name: value, name2: value2, ... }) ⇒ self

Lee los atributos de la colleción y agrega los atributos correspondientes expresados como string.

var form = $('form')
form.attr('action')             //=> leer valor
form.attr('action', '/create')  //=> definir valor
form.attr('action', null)       //=> remover atributo

before

before(content) ⇒ self

Agregar contenido al DOM antes de cada elemento individual de la colección. El contenido puede ser una cadena HTML, un nodo DOM o una matriz de nodos.

$('table').before('<p>See the following table:</p>')

css

css(property) ⇒ value
css(property, value) ⇒ self
css({ property: value, property2: value2, ... }) ⇒ self

Lee o establece las propiedades CSS de los elementos DOM. Cuando se da ningún valor, devuelve el CSS con la propiedad del primer elemento de la colección. Cuando se da un valor, establece la la propiedad en cada elemento de la colección.

var elem = $('h1')
elem.css('background-color')          // leer propiedad
elem.css('background-color', '#369')  // definit propiedad
elem.css('background-color', '')      // remover propiedad

hasClass

hasClass(name) ⇒ boolean

Hace una revisión del elemento para verificar si tiene una clase.

html

html() ⇒ string
html(content) ⇒ self
html(function(index, oldHtml){ ... }) ⇒ self

Agrega un contenido html a la coleción del elemento, este método remueve el contenido del elemento y agrega el string enviado por el método.


$('.comment p').html(function(idx, oldHtml){
  return oldHtml.replace(/(^|\W)@(\w{1,15})/g,
    '$1@<a href="http://twitter.com/$2">$2</a>')
})

next

next() ⇒ collection
next(selector) ⇒ collection [v1.0]

Obtiene el siguiente elemento filtrado por el selector de cada elemento de la colección.

$('dl dt').next()   //=> the DD elements

prev

prev() ⇒ collection
prev(selector) ⇒ collection [v1.0]

Obtiene el elemento previo filtrado por el selector de cada elemento de la colección.

removeClass

removeClass([name]) ⇒ self
removeClass(function(index, oldClassName){ ... }) ⇒ self

Remueve una clase específica de la colección del elemento expresada como un string en el llamado al método.

show

show() ⇒ self

Restaura el valor por defecto de una colección de elementos que estuvo definida en hide.

hide

hide() ⇒ self

Oculta la colección del elementos del DOM y los puedes mostrar con show.

Manejadores de Eventos

bind 🐶🔫 

bind(type, function(e){ ... }) ⇒ self
bind({ type: handler, type2: handler2, ... }) ⇒ self

Adjunta un controlador de eventos a los elementos.

unbind 🐶🔫 

unbind(type, function(e){ ... }) ⇒ self
unbind({ type: handler, type2: handler2, ... }) ⇒ self

Remueve el manejador de eventos.

Ajax

$$.ajax 

$$.ajax(options) ⇒ XMLHttpRequest
  • type: POST || GET
  • url: destination
  • success: function

De forma predeterminada, los eventos Ajax se disparan en el objeto del documento. Sin embargo, si el contexto de una solicitud es un nodo DOM, los eventos se activan en ese nodo.



$$.ajax({
  type: 'GET',
  url: '/projects',
  success: ({ function() })
  })

Templating

$$.template

tmpl = $$.template(tmpl) ⇒ string
tmpl.render(data) ⇒ tmpl

Agregar un template al objecto definiendo el renderizado de los datos mediante llaves. Con esto podemos iterar y condicionar el renderizado.



tmpl = new $$.template("Hello {{=name}}");
tmpl.render({name: "World!"});
